Github Flowとは

参加する開発者全員がGit/GitHubの経験を持っているようであれば、 各自の経験則で1つのブランチにデータを集約して作業を進めていくことができる。 リーダーとなるマスターユーザがリモートリポジトリを提供し、コードの調整管理を行う。 このような開発の流れを「GitHub Flow」と言う。このグループワークを実際に作業してみる。

Git Flowとは

GitHubFlowの場合、自分の好きなタイミングでリモートリポジトリにプッシュしていくため、 共有するデータの品質が時間とともに劣化していく。 そのため現場ではGitが推奨する共通のルールに倣って、ブランチの切り出し方を決めておくことで、 新人でもそのルールを知っていれば、すぐにプロジェクトにアサインできるようになる。

プルリクエストとは

データを受け付ける際は、ブランチからの「プルリクエスト」からでないと受け付けられない。 マスターがブランチ経由でファイルの取り込みを管理することで、招待メンバーの早い アクセス優先の競合を避けられるようになる。
